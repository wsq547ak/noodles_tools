# Tiny Style Guide

本规范用于统一 `tiny` 项目的前端视觉语言，当前版本以 Apple 风格为参考：克制、清晰、轻盈，强调内容层级与界面质感，而不是夸张装饰。

## 1. 设计原则

- 页面优先传达“安静、可靠、精确”的感受。
- 用留白、字号、字重和分组建立层级，不依赖强烈颜色和厚重阴影。
- 组件外观保持简洁，交互反馈要明确但不跳脱。
- 优先做稳定、长期可复用的系统化样式，而不是单页面特效。

## 2. 颜色系统

### 基础色

- 背景色：`#f5f5f7`
- 强背景：`#ffffff`
- 主文字：`#101828`
- 次文字：`#667085`
- 分割线：`rgba(15, 23, 42, 0.08)`

### 强调色

- 主强调色：`#2563eb`
- 强调深色：`#1d4ed8`
- 错误色：`#b42318`

### 使用规则

- 整个页面只保留一个主强调色，不混用多种高饱和颜色。
- 不使用纯黑 `#000000`，统一使用偏柔和深色。
- 大面积背景保持浅色与低对比，交互焦点通过边框、轻投影和色彩聚焦表达。

## 3. 字体系统

### 字体栈

```css
font-family:
  "SF Pro Display",
  "SF Pro Text",
  "PingFang SC",
  "Helvetica Neue",
  Helvetica,
  Arial,
  sans-serif;
```

### 规则

- 标题使用更紧的字距和更高字重，避免过度放大。
- 正文以 `15px` 到 `17px` 为主，行高保持在 `1.6` 到 `1.75`。
- 辅助说明和标签用较浅颜色，不通过过小字号制造层级。

### 推荐层级

- 一级标题：`clamp(3.2rem, 6vw, 5.8rem)`，`letter-spacing: -0.06em`
- 二级标题：`1.4rem` 到 `1.6rem`
- 正文：`1rem` 到 `1.05rem`
- 标签/辅助信息：`0.78rem` 到 `0.9rem`

## 4. 布局系统

- 页面容器优先使用 `width: min(1320px, calc(100vw - 40px))`。
- 主内容区采用两栏布局，但在 `1080px` 以下必须折叠为单栏。
- 卡片和面板之间的标准间距使用 `24px`。
- 移动端页面外边距优先使用 `24px`，避免贴边。

### 响应式原则

- `desktop`: 可以使用不对称布局
- `tablet`: 转为单列或弱不对称
- `mobile`: 强制单列，按钮和输入框尽量通栏

## 5. 面板与容器

### 面板样式

```css
border: 1px solid rgba(15, 23, 42, 0.08);
background: linear-gradient(180deg, rgba(255, 255, 255, 0.92), rgba(255, 255, 255, 0.8));
box-shadow:
  inset 0 1px 0 rgba(255, 255, 255, 0.7),
  0 24px 60px rgba(15, 23, 42, 0.06);
backdrop-filter: blur(24px);
```

### 圆角规则

- 主面板：`28px` 到 `36px`
- 次级区块：`20px` 到 `28px`
- 按钮：`999px`
- 输入框：`20px`

### 规则

- 阴影必须轻，不做发光效果。
- 面板可以有轻微玻璃感，但不能像传统玻璃拟态那样强反射、强透明。
- 只有确实需要层级区分时才使用卡片，不要整页堆卡片。

## 6. 表单规范

### 输入框

- 标签必须在输入框上方。
- 输入框内边距建议 `14px 16px`。
- 聚焦态使用浅蓝边框和外环：

```css
border-color: rgba(37, 99, 235, 0.32);
box-shadow: 0 0 0 4px rgba(37, 99, 235, 0.08);
```

### 文案规则

- Label 说明字段含义
- Helper Text 说明规则或格式
- Error Text 放在输入区块下方

## 7. 按钮规范

### 一级按钮

- 蓝色实底
- 白色文字
- 可有非常轻的纵向渐变

### 二级按钮

- 白底或半透明浅底
- 1px 细边框
- 深色文字

### 交互反馈

- `hover`: `translateY(-1px)`
- `active`: `scale(0.98)`
- `disabled`: `opacity: 0.45`

禁止使用外发光、霓虹色、夸张缩放。

## 8. 状态设计

所有工具页至少要提供以下状态：

- 默认态
- 加载态
- 成功态
- 错误态
- 空状态

### 规范

- 加载态优先用按钮内文案变化或骨架，不优先用旋转 loading。
- 成功态和错误态使用轻背景 + 细边框提示。
- 空状态需要告诉用户下一步该做什么。

## 9. 文案风格

- 直接、准确、低情绪化。
- 少用营销化表达。
- 少用感叹句和修饰词。
- 中文界面尽量统一术语，例如“样本”“期望结果”“提取结果”“测试文本”。

## 10. 当前项目落地建议

### `regseed`

- 保持双栏：左侧输入样本，右侧验证结果。
- Hero 只承担说明与状态概览，不放冗余按钮。
- 结果区每条测试记录同时展示原文与输出，方便比对。

### 后续页面

- `piczip` 可以沿用同一套背景、圆角、按钮和表单系统。
- 新增页面时优先复用相同的颜色变量和面板结构。
- 如果后面引入 Tailwind，先把这些 token 抽成全局 design tokens 再迁移。
